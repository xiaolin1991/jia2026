使用说明（本地/手机）

1) 这是纯前端页面：monitor.html（监控）与 settings.html（设置）。
2) 手机浏览器/微信要打开摄像头：必须运行在 HTTPS 或 localhost。

本地调试建议：用任意静态服务器启动（例如 VSCode Live Server）。

后续我会继续接入 MediaPipe Pose 来做坐姿识别与提醒。

常见部署方式（让手机可用摄像头）

1) 把 posture-guardian 目录部署到任意静态托管：例如 Vercel / Netlify / GitHub Pages。
2) 确保最终访问地址是 https:// 开头（HTTPS）。
3) 用手机浏览器或微信打开 monitor.html 页面。

兼容性说明（华为/微信/vivo/oppo）

1) Android 主流浏览器（华为浏览器、OPPO/vivo 浏览器、Chrome）一般支持 getUserMedia 打开摄像头。
2) 微信内置浏览器（X5 WebView）通常也可以，但常见限制是：
   - 语音播报可能必须由“用户点击”触发。若语音不响：先点一次“播放一句”，再开始。
   - 首次权限弹窗如果点了拒绝，需要去系统设置里重新允许。
3) 如果出现“Pose 库加载失败”，一般是网络或 CDN 不可达，换网络/稍后再试。

排查清单

1) 是否 HTTPS：不是 HTTPS，手机一定打不开摄像头。
2) 是否允许权限：浏览器地址栏/微信权限管理里是否允许相机。
3) 是否被拦截：某些网络会拦截第三方 CDN，导致坐姿识别库加载失败。
4) 光线与距离：确保上半身在画面内，光线充足，坐在桌前正对镜头。
